# 📰 Fine-tuning Qwen2.5-1.5B-Instruct для генерации Telegram-постов

> **LoRA-дообучение** компактной инструкционной LLM под стилистику городских и скандальных Telegram-каналов с помощью [Unsloth](https://github.com/unslothai/unsloth).

---

## 🚀 1. Обзор проекта и быстрый старт

### Что это?

Это учебно-портфолио проект по **Supervised Fine-Tuning (SFT)** модели **Qwen2.5-1.5B-Instruct**. Цель — научить модель генерировать короткие посты в формате Telegram-каналов: от нейтрально-информативных городских новостей до агрессивного кликбейтного контента.

Ключевая идея: вместо полного дообучения всех весов модели используется **LoRA-адаптер** — лёгкая надстройка, которая «запоминает» стиль и формат, не затирая базовые знания LLM.

**Стек:** PyTorch · Transformers · TRL · PEFT · Unsloth · Hugging Face Hub

## 📁 2. Описание файлов и запуск

### Структура проекта

| Файл / папка | Назначение |
|---|---|
| `train.py` | Скрипт обучения LoRA-адаптера |
| `inference.py` | Сравнительный инференс: базовая модель vs LoRA |
| `dataset.jsonl` | Унифицированный обучающий датасет (формат chat messages) |
| `my_model/` | Локальные веса Qwen2.5-1.5B-Instruct |
| `lora_qwen_telegram/` | Сохранённые LoRA-веса после обучения |
| `outputs_type1.txt` | Результаты тестов — **Городской канал** |
| `outputs_type2.txt` | Результаты тестов — **Скандальный канал** |

---

### `train.py` — обучение

Скрипт выполняет полный пайплайн SFT:

1. **Загрузка модели** — `FastLanguageModel.from_pretrained()` с 4-bit квантизацией (QLoRA) для экономии VRAM.
2. **Настройка LoRA** — rank `r=16`, alpha `32`, target modules: attention + MLP-проекции.
3. **Подготовка датасета** — чтение `dataset.jsonl`, применение chat-шаблона Qwen 2.5.
4. **Тренировка** — `SFTTrainer` (TRL), 3 эпохи, effective batch size = 8.
5. **Сохранение** — LoRA-веса и токенизатор в папку `lora_qwen_telegram/`.

```bash
MODEL_PATH="./my_model" DATASET_PATH="dataset.jsonl" python train.py
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| `MODEL_PATH` | `./my_model` | Путь к базовой модели |
| `DATASET_PATH` | `dataset.jsonl` | Путь к обучающему датасету |

---

### `inference.py` — оценка качества

Скрипт загружает базовую модель + LoRA-адаптер и прогоняет **10 тестовых запросов** (5 на каждый тип канала). Для каждого вопроса генерируются **два ответа**:

- ❌ **Без LoRA** — `model.disable_adapter()`, базовая модель.
- ✅ **С LoRA** — дообученный адаптер активен.

Результаты записываются в `outputs_type1.txt` и `outputs_type2.txt`.

```bash
MODEL_PATH="./my_model" LORA_PATH="./lora_qwen_telegram" python inference.py
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| `MODEL_PATH` | `./my_model` | Путь к базовой модели |
| `LORA_PATH` | `./lora_qwen_telegram` | Путь к LoRA-адаптеру |

## 📊 3. Анализ результатов

> Результаты получены на **zero-shot** запросах — темы и формулировки **не встречались** в `dataset.jsonl`.

---

### 🔍 Общие выводы: ДО vs ПОСЛЕ

| Критерий | Базовая модель (ДО) | LoRA-модель (ПОСЛЕ) |
|---|---|---|
| **Длина текста** | Часто многословна, уходит в простыни | 1–2 коротких абзаца, как в промпте |
| **Формат** | Хаотичный: `###`, хештеги, markdown | Чистый Telegram-пост |
| **Стиль** | Обобщённый, «нейросетевой» | Узнаваемый стиль канала |
| **Язык** | Иногда переключается (китайский, англ.) | Стабильно русский |
| **Эмодзи** | Случайные, избыточные | Одно тематическое в начале |
| **Релевантность** | Галлюцинирует, уходит от темы | Держится в рамках запроса |

**Базовая модель** формально «понимала» задачу, но отвечала как универсальный ассистент: добавляла лишние хештеги, советы читателям, морализаторство или вообще генерировала текст на другом языке. Формат Telegram-поста не соблюдался.

**LoRA-версия** научилась **чётко следовать инструкции**: короткий заголовок с эмодзи, 1–2 абзаца фактуры, при необходимости — строка с источником. Модель уловила не отдельные фразы из датасета, а **паттерн подачи материала**.

---

### 🧪 Zero-shot возможности (Out-of-domain)

> **Важно:** все 10 тестовых вопросов в `inference.py` были **придуманы вручную** и **не содержатся** в `dataset.jsonl`.

Примеры out-of-domain тем из тестов:

- BIM-модели для мониторинга ремонта МКАД
- Кластер H100 в промзоне «Руднево»
- Блокировка протокола VLESS Роскомнадзором
- LangGraph и сломанная конфигурация GPU-кластера

Ни одна из этих формулировок не встречается в обучающих данных. Тем не менее LoRA-модель:

- ✅ генерирует текст в нужном **формате** (эмодзи, абзацы, источник);
- ✅ переключает **тональность** в зависимости от системного промпта;
- ✅ использует **доменную лексику** (BIM, H100, VLESS, LangGraph) без вызубривания.

Это доказывает, что модель **не переобучилась (overfitting)** на конкретные примеры, а **обобщила стилистические паттерны** и применяет их на новых, незнакомых данных.

---

### 🎭 Влияние системного промпта

Один и тот же LoRA-адаптер, но **разный system prompt** — принципиально разный результат. Модель жёстко слушается инструкции редактора.

#### 🏙️ Городской канал (`outputs_type1.txt`)

**Промпт:** нейтрально-информативный стиль, эмодзи в начале, 1–2 абзаца, источник в конце.

| # | Базовая модель | LoRA |
|---|---|---|
| 1 | `### 🚪` + формальный текст про BIM | `🏗️` + 2 коротких абзаца про пилот на МКАД |
| 2 | Хештеги `#Нейросети #Исследования`, путает H100 с компанией | `🔹` + сухая новость про кластер в «Руднево» |
| 3 | Драматизация: «авария на кухне», «зловоние», китайский символ 宁静 | `🚁` + бытовая заметка про Жигули во дворе |
| 4 | Морализаторство: «мечтают о карьере в AI» | `👷‍♂️` + статистика по вакансиям |
| 5 | **Текст на китайском** про ДТП | `🥲` + 2 строки + источник `Москвач` |

> LoRA-модель стабильно выдаёт **короткие городские заметки** с одним эмодзи и нейтральным тоном — ровно то, что требует промпт.

#### 🔥 Скандальный канал (`outputs_type2.txt`)

**Промпт:** агрессивный кликбейт, гиперболизация, шокирующее начало, ссылки на источники.

| # | Базовая модель | LoRA |
|---|---|---|
| 1 | Морализаторство + `#РаботаСОК`, обрывается на полуслове | Переработки разработчиков, ссылки на Mashable и The Verge |
| 2 | **Полный off-topic:** алкоголь, водительские права, `#AlcoholIsTheLaw` | «финик» и «лоер» — юмористический кликбейт про суд |
| 3 | Агрессия без фактов, «мы все равно будем использовать прокси» | «убийственное падение прокси», выдуманный «пользователь Дмитрий» |
| 4 | Восторженное «ты просто великолепен! 🌟✨» | Сухой скандал: отчислили → стал инженером → зарабатывает больше |
| 5 | Цитата из фильма «Совпадения», `#Политика_допинга` | «20 миллионов рублей», «микроконтексты» LangGraph — желтушная драматизация |

> При промпте скандального канала LoRA-модель **переключается в другой регистр**: добавляет вымышленные источники, гиперболизирует, начинает с провокационного утверждения — без единого эмодзи и хештега.

**Вывод:** системный промпт — главный «переключатель режима». LoRA запомнила **оба стиля** и выбирает нужный по инструкции в `system`-сообщении.

## 📦 4. Заметка о данных

В ранних итерациях пайплайна обучающие примеры хранились в отдельных файлах:

- `inputs_type1.txt` — сырые факты для городского канала
- `inputs_type2.txt` — сырые факты для скандального канала

**В текущей версии проекта эти файлы не используются.**

Все данные сведены в единый **`dataset.jsonl`** в формате chat messages:

```json
{
  "messages": [
    {"role": "system", "content": "Ты — редактор ..."},
    {"role": "user", "content": "Сырой факт или запрос"},
    {"role": "assistant", "content": "Готовый пост в нужном стиле"}
  ]
}
```

Такой формат:

- ✅ напрямую совместим с `SFTTrainer` и chat template Qwen 2.5;
- ✅ включает system prompt в каждый пример — модель учится переключать стили;
- ✅ упрощает пайплайн: один файл вместо нескольких промежуточных.

## 🧠 5. Методология и выбор параметров

### Почему Unsloth?

| Преимущество | Что это даёт |
|---|---|
| **2× ускорение** | Обучение и инференс быстрее стандартного HF + PEFT |
| **Экономия VRAM** | 4-bit QLoRA + gradient checkpointing → обучение на домашней GPU (8–12 GB) |
| **Нативная интеграция** | `FastLanguageModel`, оптимизированный `for_inference()`, `disable_adapter()` |
| **Совместимость** | Полная поддержка Qwen 2.5 chat template из коробки |

Unsloth позволил пройти полный цикл **download → train → evaluate** на одной видеокарте без облачных инстансов.

---

### Почему LoRA: `r=16`, `lora_alpha=32`?

```python
model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    lora_alpha = 32,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj",
                      "gate_proj", "up_proj", "down_proj"],
    lora_dropout = 0,
)
```

| Параметр | Значение | Обоснование |
|---|---|---|
| `r` (rank) | **16** | Достаточная ёмкость для запоминания **стиля, сленга и формата** Telegram-постов. `r=8` — маловато для двух разных «редакционных голосов»; `r=32+` — избыточно, растёт риск overfitting и VRAM |
| `lora_alpha` | **32** | Классическое правило `alpha = 2 × r`. Масштабирует вклад LoRA-обновлений, обеспечивая стабильную сходимость |
| `lora_dropout` | **0** | При маленьком датасете dropout может мешать выучить стиль; регуляризация обеспечивается `weight_decay=0.01` |
| `target_modules` | **7 проекций** | Attention (Q/K/V/O) + MLP (gate/up/down) — стандартный набор для language adaptation |

---

### Почему `batch_size=2` и `gradient_accumulation_steps=4`?

```python
TrainingArguments(
    per_device_train_batch_size = 2,
    gradient_accumulation_steps = 4,
    # effective batch size = 2 × 4 = 8
)
```

| Параметр | Значение | Зачем |
|---|---|---|
| `per_device_train_batch_size` | **2** | Максимум, который помещается в VRAM при `max_seq_length=2048` + 4-bit + LoRA |
| `gradient_accumulation_steps` | **4** | Эмуляция batch size = **8** без OOM |
| Effective batch size | **8** | Стабильный градиент для SFT; достаточно для ~600 примеров × 3 эпохи |

> **Формула:** `effective_batch = per_device_batch × gradient_accumulation × num_gpus`

Это компромисс между **стабильностью обучения** и **ограничениями домашней GPU**.

---

### Прочие гиперпараметры

| Параметр | Значение | Комментарий |
|---|---|---|
| `num_train_epochs` | 3 | Достаточно для усвоения стиля без переобучения |
| `learning_rate` | 2e-4 | Стандарт для LoRA SFT |
| `warmup_steps` | 10 | Плавный старт оптимизатора |
| `optim` | adamw_8bit | Экономия памяти |
| `max_seq_length` | 2048 | Запас под длинные посты |
| `load_in_4bit` | True | QLoRA — обучение 1.5B модели на ~6 GB VRAM |

---

### Итог: качество без catastrophic forgetting

Комбинация **LoRA (r=16) + 4-bit QLoRA + 3 эпохи + effective batch 8** дала:

- ✅ **Стилистическую адаптацию** — модель пишет как редактор Telegram-канала
- ✅ **Гибкость по промпту** — переключение между городским и скандальным стилем
- ✅ **Zero-shot генерацию** — работает на темах, которых не было в датасете
- ✅ **Сохранение базовых знаний** — модель не «сломалась», понимает доменную лексику (BIM, RAG, VLESS, LangGraph)
- ✅ **Отсутствие catastrophic forgetting** — базовые веса заморожены, обновляется только LoRA-адаптер (~0.5% параметров)

> LoRA — это не замена модели, а **тонкая настройка «голоса»** поверх сохранённого интеллекта Qwen2.5-1.5B-Instruct.
